使用 **`hig-review` skill** 执行一次完整的「HIG 滚动基线审查」（流程同 `review` 命令：
Step 0 先确认产品类型 / 输入 / 规则源，再 detect → 加载规则 → 评估 → 出报告），**并在整个
审查过程中纳入下面这条人工补充指令**作为额外的关注重点 / 附加规则：

> 补充指令：**$ARGUMENTS**

要求：

- 把补充指令当作高优先级的审查侧重点（例如指定平台、深色模式、无障碍对比度、特定组件或团队
  自定规范），在评估和报告里优先覆盖；但仍以 HIG 基线为依据，不要凭空编造规则。
- 在报告开头注明「本次审查附加了人工补充指令：……」，让结论可追溯。
- 若上面的补充指令为空，**先询问用户**希望补充哪些关注点 / 额外规则，拿到后再开始审查。
