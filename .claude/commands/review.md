使用 **`hig-review` skill** 对项目进行一次完整的「HIG 滚动基线审查」，严格遵循该 skill
`SKILL.md` 的工作流：

1. **Step 0 — 先问后审**：用面向用户的话确认 ①产品类型（iPhone / iPad / Mac /
   Apple Watch · TV · Vision Pro / 网页 · 数据大屏）②你给的是什么（代码 / UI 图 /
   UX 文档 / 设计稿，Figma 链接等，能明显推断的就直接说明假设）③用「本地规范(更快)」
   还是「联网获取最新规范(稍慢)」。若选「本地」但基线尚未构建（`rules/index.json` 不存在），
   先展示首次同步提示再行动：
   > 首次使用「离线审查」需先把 Apple HIG 同步到本地基线库（联网抓取一次，约 15–20 分钟，
   > 视网络情况）；完成后审查全程离线、更快。现在开始同步，还是改用「联网获取最新规范」？

   用户选同步则运行构建（等同 `baseline-init` 命令），不想等就改用联网模式。
2. **Step 1–4** — 按 skill 的 detect → 加载规则 → 评估 → 出报告流程执行，最终产出标题为
   **「HIG 滚动基线审查报告」** 的报告（汇总表 + 逐条 issue：位置 + 问题 + 影响 + 修复 +
   依据的指南名称与链接）。

审查目标：**$ARGUMENTS**

- 若上面给了路径 / 文件 / 链接，就以它为审查对象（单个图片 / 文档 / Figma 链接按 skill
  对应 playbook 处理；项目目录先跑 `detect_inputs.py`）。
- 若为空，按 Step 0 询问后再开始。
